# Git Commit & Push

변경사항을 분석하여 의미있는 커밋 메시지를 작성하고, 커밋 및 푸시를 수행합니다.

## 워크플로우

### 1. 변경사항 확인

```bash
git status --short
```

변경사항이 없으면 "커밋할 변경사항이 없습니다"라고 알리고 종료.

### 2. 변경 내용 분석

```bash
# 수정된 파일 diff 확인
git diff --stat

# 상세 내용이 필요하면
git diff --name-status
```

### 3. 커밋 타입 결정

변경 내용에 따라 적절한 타입 선택:
- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 변경
- **refactor**: 코드 리팩토링 (기능 변경 없음)
- **style**: 코드 포맷팅, 세미콜론 등
- **test**: 테스트 추가/수정
- **chore**: 빌드, 설정 파일 등 기타 변경

### 4. 커밋 메시지 작성

형식 (Conventional Commits):
```
<type>: <간결한 설명>

- 세부 변경사항 1
- 세부 변경사항 2
```

예시:
```
feat: 사용자 프로필 편집 기능 추가

- 프로필 이미지 업로드
- 닉네임 변경 기능
- 자기소개 입력 필드
```

### 5. 커밋 실행

```bash
git add .
git commit -m "커밋 메시지"
```

### 6. 푸시 실행

```bash
git push
```

푸시 실패 시 (upstream 미설정):
```bash
git push -u origin $(git branch --show-current)
```

### 7. 결과 보고

커밋 해시와 푸시 결과를 사용자에게 알림.

## 주의사항

- IDE/에디터 설정 폴더 변경은 커밋 메시지에서 생략 (`.vscode/`, `.idea/` 등)
- 민감한 파일(`.env`, `credentials.json` 등)이 포함되어 있으면 경고
- 대량 변경(50개 이상 파일) 시 사용자 확인 요청
- `node_modules/`, `dist/`, `build/` 등 빌드 산출물은 커밋하지 않음
